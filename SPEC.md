# AI Murder Mystery Detective

## What it is (1 sentence)
An AI-driven web game where players interrogate suspects in natural language to solve a uniquely generated murder mystery before time runs out.

## Track
[ ] Internal Tool   [ ] Fintech Mini-App   [ ] Crypto/Web3   [x] Free-form

## Target user (1 sentence)
Casual gamers and fans of detective fiction who enjoy mystery novels, films, or board games like Clue.

## The user's job-to-be-done (1 sentence)
Piece together who committed a murder — and why — purely through smart questioning, observation, and reasoning, with no hand-holding.

## Must-have features (all IMPLEMENTED ✅)
1. **Google Auth + Coin Economy** ✅ — User logs in with Google, receives 1 000 starter coins, cannot play without login, coin balance persists across sessions and is transactional (never goes below 0).
2. **Case Generation (Easy / Medium / Hard)** ✅ — Selecting a difficulty spends coins (50 / 100 / 200), AI generates a unique case file (victim, crime scene, 3–6 suspects + 2–3 witnesses) from a pool of 24 diverse settings with culturally varied character names. Each suspect/witness carries a `gender` field for voice selection. Timer starts after player reads the case file brief (25 / 35 / 55 min).
3. **Natural-Language Suspect Interrogation** ✅ — Player types any question to any suspect or witness; each character responds in a consistent voice reflecting their private truth, secrets, and personality. AI is injected with the full case JSON server-side per turn.
4. **Hint System** ✅ — One hint per case; returns a subtle first-person nudge ("Something about X doesn't sit right…"); `HintAlreadyUsedError` thrown on second request.
5. **Accusation & Reveal** ✅ — Player selects a suspect and submits; backend validates, calculates coins, returns narrative reveal. Timer expiry auto-submits as a loss. Stats written to `localStorage`.

## Tech stack
- Backend: TypeScript 5.4 + NestJS 10 (Node 22) — `backend/`
- DB: Neon Postgres (serverless) via Sequelize ORM + Sequelize migrations
- Frontend: Next.js 15 App Router (React 19) + Tailwind CSS v3 + shadcn/ui — `frontend/`
- Auth: Google OAuth handled by NestJS backend; JWT issued to frontend
- AI: Groq — model `llama-3.3-70b-versatile` (groq-sdk, server-side only)
- Audio: Web Audio API (procedural SFX + canvas-synced thunder); Web Speech API (TTS output + speech-to-text input); HTMLAudioElement (looping mystery-thriller score with route-aware on/off)
- Animations: framer-motion (spring physics, AnimatePresence, stagger)
- Monorepo: pnpm 9 workspaces (`frontend/`, `backend/`)
- Platform-specific feature: Countdown timer with auto-submit on expiry; client-side stats in localStorage

## Architecture (5 lines max)
Frontend is a Next.js App Router SPA (Tailwind dark noir theme) that communicates exclusively with the NestJS backend over REST.
NestJS backend owns all logic: Google OAuth callback, JWT auth, four Groq-backed modules (case generation, interrogation, hint, verdict).
Each suspect's private truth, `alibi_is_true`, and `will_crack_if` are stored in Postgres per session and never returned to the frontend.
Groq generates the full case JSON once at game start; each interrogation turn injects the full case context server-side — the frontend never holds private data.
Coin balance and game state persist in Postgres; Railway hosts the backend, Vercel hosts the frontend.

## Out of scope
- Real-money coin purchases (coins are earned by play only)
- Multiplayer or co-op investigation
- Mobile native app (web-responsive only)
- Cross-device stats sync (stats live in localStorage)

## Risks & mitigations
1. **AI response consistency** — Mitigated by storing the full case JSON server-side and injecting it into every Groq prompt. Suspect cannot contradict themselves across turns.
2. **Cost blow-up** — Groq llama-3.3-70b is free-tier friendly; no streaming. Each turn is a self-contained REST call.
3. **TTS voice quality** — Depends on the user's OS/browser voice list. Mitigated by a graceful fallback chain: gender-specific voice → neutral best voice → first available English voice.
4. **Image generation latency** — Crime scene image shows a placeholder async. Does not block game start.

## How I'll demo it (5 steps)
1. Open landing page — atmospheric canvas with leaves, lightning, thunder. Click "Begin Investigation".
2. Log in with Google → dashboard shows coin balance (1 000), personalised greeting, and animated stats panel.
3. Select Medium difficulty (costs 100 coins) → case file book opens with narration; click "Begin Investigation" to start 35-minute timer.
4. Interrogate two suspects in the split-pane view; enable voice to hear AI responses read in character (gender-matched browser voice). Notice a contradiction in one suspect's alibi. Use the one hint to confirm.
5. Submit accusation → verdict reveal page shows cinematic narrative, coins awarded (300 on correct). Stats update in dashboard.
